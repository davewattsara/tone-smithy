New `filter/` submodule per project-structure.md with `svf.rs` holding the TPT state-variable filter (Andrew Simper's "state increment form" of Zavalishin's TPT). All four output taps share the same integrator state, so mode flips are click-free.

`FilterCutoffHz` and `FilterResonance` are smoothed continuous params on the parameter tree; `SetFilterMode` is a typed discrete event matching the `SetOscillatorWaveform` pattern. Resonance maps linearly to Q in `0.5..=25` — ringy enough to "sing" at the top without letting the integrator state diverge on bounded input.

`Voice::next_sample` now takes `&SampleParams` rather than a bare `f32` so the engine has a single per-sample fan-out point. Signal flow: osc mix × 0.25 → filter → amp envelope. The filter resets on every note-from-silence so a fresh note doesn't inherit a ringing tail.

`ParamSnapshot` gains three new fields (`filter_cutoff_hz`, `filter_resonance`, `filter_mode`); the UI is non-destructively expandable. End-to-end test: holding a saw then closing the LP cutoff to 30 Hz silences the voice to under 10 % of the open level. `no_alloc.rs` sweeps cutoff/resonance/mode inside the no-alloc scope so the per-sample `tan()` recompute path is covered.

### [12:23:00] Claude (Opus 4.7)
The engine becomes properly stereo at this milestone. `Voice::next_sample` now returns `(f32, f32)` instead of a mono `f32`; `Engine::process_stereo` forwards the pair into the interleaved output, ending M2.2-era mono duplication. The voice now holds a per-channel SVF (integrator state has to be per-channel because L and R inputs differ; cutoff/resonance still come from a single source so the two filters track).

Eleven new flat `ParamId`s (4 levels, 3 main detunes, 4 pans). Sub gets level and pan but no detune — by definition it's exactly one octave below the base pitch. Per-osc mixing inside the voice: each oscillator scaled by its smoothed level, split L/R by equal-power pan, then the per-channel sums get the slot headroom scale (1/4) before the filter. Pan law uses two sqrts rather than sin/cos — fewer flops, mathematically equivalent.

`MAIN_OSCILLATOR_COUNT` promoted from voice.rs to lib.rs as a top-level public constant so the param tree, snapshot, and future preset format agree on array sizes. Default per-osc level = 1.0, pan = 0.0 — fresh patches are about 3 dB quieter per channel than M2.2 because center equal-power gives 0.707 per side rather than the previous mono-duplicated 1.0. Accepting the drop rather than papering over with a compensation factor; the user can crank master volume.

End-to-end test: hard-pan osc1 left, mute the others, render — right channel stays under 0.01 while left exceeds 0.1. Voice tests cover stereo silence, mute-all, hard-pan routing, +1200 cents detune verified by zero-crossing count, and the equal-power L²+R²=1 identity. no_alloc test sweeps per-osc levels, detunes, and pans inside the no-alloc scope.
