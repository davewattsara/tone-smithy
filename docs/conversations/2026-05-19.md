### [12:28:20] Claude (Opus 4.7)
`UnisonOscillator` wraps up to seven `Oscillator` instances behind one shared waveform, mixing them with linear detune spread across `[-d, +d]` cents and stereo spread around the per-osc center pan. Summed output normalised by `1/sqrt(N)` so the bank's perceived loudness is roughly invariant in voice count — the analog-unison property.

Phase decorrelation: a small LCG (no allocation, audio-thread safe) randomises phases on note-on and when voice count increases. Without it the bank would comb-filter at attack from perfectly aligned starting phases.

Nine new flat `ParamId`s for the three main banks: voice count (stepped — integer counts don't benefit from smoothing), unison detune cents, unison stereo spread. Defaults: 1 voice (fresh patches sound identical to M2.3), 10 cents detune, 0.5 spread (the first crank of voices already sounds like unison). Sub stays a single `Oscillator` — by definition one octave below the base pitch, unison wouldn't add musical value there.

`equal_power_pan` moved to a new top-level `panning.rs` so both the voice's slot mixer and the unison oscillator can call it without a circular import. `MAX_UNISON_VOICES = 7` joins `MAIN_OSCILLATOR_COUNT` as a top-level public constant.

### [18:18:00] Claude (Sonnet 4.6)
Read all 23 RS source files. Two genuine bugs found and fixed; everything else checked out clean.

**Bug 1 — oscillator phase wrap (`subtractive.rs`):** Phase accumulator used `if self.phase >= 1.0 { self.phase -= 1.0; }`. If `phase_increment > 1.0` (frequency > sample rate), a single subtraction leaves phase still >= 1.0, corrupting all subsequent samples. Reachable at MIDI 127 + ±24 st pitch offset at 48 kHz (≈ 50 kHz > 48 kHz). Fixed by changing `if` to `while`.

**Bug 2 — misleading "Q" label (`app.rs`):** Status bar displayed raw 0..=1 resonance value labelled "Q". Actual Q is 0.5–25.0 after `resonance_to_q` mapping. Relabelled as "res".

False alarms (agent-flagged, manual verification cleared): envelope decay transition, `round_voice_count` NaN/infinity handling, doc-comment coverage — all were correct. The `fract(x)` helper is fine once the phase wrap is fixed (input is always in [0.5, 1.5) from a corrected phase).

The fix (listening to `IMMNotificationClient` events and reopening the stream on device change) is M13 scope. For now: plug in headphones before launching.
