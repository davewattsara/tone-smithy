### [22:14:00] **You**
> can you look for any bugs. if you find any fix them

### [22:14:48] Claude (Sonnet 4.6)
Reviewed all M6 code (mod_matrix.rs, voice.rs, voice_manager.rs, params.rs,
engine.rs, app.rs) and found three bugs:

1. **Amount not reset on dest change** (serious): when the user changed the
   destination combo, the amount from the previous dest persisted in the engine.
   This could lock the knob at max — the Knob widget only clamps backing values
   on drag, not on render, so an out-of-range backing value (e.g. 5000 for a
   ±1 resonance slot) made the knob appear stuck. Fixed: dest combo change now
   also sends ModSlotAmount=0 and clears the backing variable.

2. **ModSlotSource/Via validation mismatch** (minor): ParameterTree normalised
   invalid bus indices to Off via unwrap_or_default, but the engine fan-out used
   if-let and skipped the VoiceManager update entirely on invalid input. This left
   VoiceManager with the old source while the snapshot showed Off. Fixed: engine
   fan-out now uses unwrap_or_default to match the tree.

3. **Dead was_enabled variable** in the checkbox handler — removed.

All 163 tests still pass; fmt and clippy clean.

**Commits:** `bae5158` — Fix three M6 bugs
