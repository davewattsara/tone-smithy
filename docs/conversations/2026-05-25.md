Drafted the M7 plan at `docs/planning/06-implementation/m07-plan.md`.
Key design decisions:

- **Two-slot voice:** each Slot is a struct with both subtractive and
  FM state plus a mode-tag; only the active bank advances each sample.
  Keeps everything stack-allocated.
- **Operator model:** phase, ADSR, integer + fine ratio, level. Op 3
  has self-feedback (DX7 convention). Evaluation order op 3 → 2 → 1 → 0.
- **8 algorithms** defined as a `const ALGORITHMS: [Algorithm; 8]`
  table of `mod_sources: [u8; 4]` bitmasks + `is_carrier: [bool; 4]`.
- **Oversampling:** always-on 2× with a 31-tap symmetric FIR half-band
  (simpler than conditional, fixed CPU cost).
- **Sub-milestones M7.0–M7.5:** slot refactor (no behaviour change) →
  operator + algorithm types → FmBank-as-slot-mode → param bus + UI →
  oversampling → close-out.

### [HH:MM:SS] Claude (Opus 4.7)
Implemented M7.1 — FM operator + 8 starter algorithms in a new
`crates/synth-engine/src/fm.rs`, not yet wired into Voice.

Operator owns: phase (0..1), ADSR, integer ratio (0..=15), fine cents
(±100), level (0..=1), and feedback fields (one-sample delayed). Output
is `sin(2π × (phase + mod_in).rem_euclid(1.0)) × env × level`. Ratio 0
maps to ratio 1 for M7 (fixed-frequency deferred to v1.2).

`Algorithm` is `{ mod_sources: [u8; 4], is_carrier: [bool; 4] }`. The
bit-j-set-in-mod_sources[i] convention encodes "op j modulates op i";
self-reference (j == i) means feedback. 8 algorithms hardcoded as
`pub const ALGORITHMS: [Algorithm; 8]` — algorithm 1 is the canonical
DX7 stack, algorithm 7 is pure additive, others in between.

`FmBank` evaluates op 3 → 2 → 1 → 0 each sample, stages outputs in a
scratch buffer, sums carriers for the slot's mono output.

8 unit tests cover the algorithm table shape, phase wrapping, additive
algorithm produces clean sine, single-carrier stack stays bounded, max
feedback never goes non-finite, ratio 0 ≡ ratio 1, and fine cents
shifts frequency. All 178 workspace tests pass; no-alloc still clean.
