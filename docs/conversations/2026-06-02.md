Read all source files across synth-engine, synth-ui, synth-host, and synth-app. Found two real bugs (the agent also flagged several non-bugs or already-guarded paths):

**Bug: Arp gate-off respects sustain pedal** — Arp NoteOff events were routed through `voices.note_off()`, which defers releases while the sustain pedal is held. With sustain held and arp running, every step's gate-off was deferred instead of firing, causing notes to pile up indefinitely. Added `release_note_immediate()` to `VoiceManager` (bypasses deferral, also clears any stale deferred entry) and routed all arp-generated NoteOff events through it.

**Bug: `set_cc` out-of-bounds on malformed MIDI** — `cc_values` is `[f32; 128]` but `cc` is `u8` (0-255). A CC byte outside 0-127 would index out of bounds. Added a `len()` bounds guard; standard MIDI devices never send such bytes but it costs nothing to defend.
