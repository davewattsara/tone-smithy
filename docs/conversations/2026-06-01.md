### [session-continued] Claude (Sonnet 4.6)

Fixed asymmetry silence bug. Root cause: old `shape()` added the bias before the drive multiply
(`(x + asym) * drive`), so at asym=1 and drive=20 the bias became +20 units into tanh, fully
saturating it. The DC removal then subtracted the raw asymmetry value (1.0) from a tanh output
of ~1.0, cancelling the signal. Fix: inject bias after drive (`x * drive + asym`) so it stays
bounded to ±1 regardless of drive, and remove DC as `tanh(asym)` not the raw value.

### [session-continued] User

i don't think the compensation gain is working.

---

### [session-continued] Claude (Sonnet 4.6)

Fixed output_gain formula. Old: `1/tanh(drive)` normalises for a full-scale 1.0 input, so
typical synth signals peaking at ~0.5 got ~4 dB louder as drive increased. New:
`0.5/tanh(0.5*drive)` anchors at 0.5 reference amplitude — a 0.5-peak signal produces a
0.5-peak output at any drive setting.

### [session-continued] Claude (Sonnet 4.6)

Fixed ping-pong. Both buffers previously received the direct input, so for a centred signal
`delayed_l == delayed_r` and swapping feedback made no difference. In ping-pong mode, input
now enters only the L buffer (as a mono sum); R receives no direct input and fills solely from
the L→R feedback crossover. Echo 1 is L-only, echo 2 is R-only, alternating each cycle.
