Built M1 in six commits on `development`:

##### C1 ‚Äî engine sine + ADSR + 1 voice playing through cpal

New `synth-engine` modules: `events.rs` (`EngineEvent` enum, NoteOn/NoteOff for now), `oscillator.rs` (phase-accumulating sine + `midi_note_to_hz`), `envelope.rs` (linear-segment ADSR with idle/attack/decay/sustain/release state machine, per-sample step recomputed on setter or phase change), `voice.rs` (one osc + one ADSR + currently-held MIDI note; out-of-order note-offs ignored), `engine.rs` (top-level with `process_stereo` and `handle`). `synth-host::audio` got a new `start_with_engine` that moves the engine into an f32 cpal callback with a pre-allocated `MAX_BLOCK_SIZE * 2` scratch buffer, plus a `default_output_format()` query so the app can build the engine at the device sample rate before starting the stream. `synth-app` fires one hardcoded `NoteOn(69)` at startup so the path is audible end-to-end.

##### C2 ‚Äî naive saw waveform + waveform-select event

`Waveform { Sine, Saw }` on the oscillator, default Sine. Saw is the naive ramp `(phase / PI) - 1.0`; PolyBLEP anti-aliasing is the M2 deliverable. New `EngineEvent::SetOscillatorWaveform { waveform }` routed via `Voice::set_waveform`. Discrete param, latched at block boundary because the engine drains events before processing samples (design-patterns.md ¬ß2.7). The startup demo note switched to saw.

##### C3 ‚Äî parameter bus (SPSC events + ArcSwap snapshot)

New engine `params` module: `ParamId` (PitchOffsetSemis, AmpReleaseSecs) and `ParamSnapshot` (pitch offset, amp release, waveform, voice_active). New `EngineEvent::ParameterChange { id, value }`. Engine mirrors snapshot fields locally so `snapshot()` returns by value with zero allocation; the audio callback wraps in `Arc::new` once per block ‚Äî that allocation is the intentional cost called out by design-patterns ¬ß1.4 and lives outside the DSP hot path the C6 test will guard. New `synth-host::param_bus` with `EngineEventSender`/`EngineEventReceiver` (logged-drop SPSC wrapper at the ¬ß2.2 default 4096 capacity) and `SnapshotSlot = Arc<ArcSwap<ParamSnapshot>>`. `start_with_engine` now drains the queue at the top of each block and stores a fresh snapshot at the end. `synth-app` queues the demo events through the bus so the queue-drain path runs from the first callback.

##### C4 ‚Äî UI sliders + bus relocation

The ¬ß1.6 plan has the composition root construct raw `Sender` / `ArcSwap` types and pass them to both `synth-ui` and `synth-host`. `synth-ui` depending on `synth-host` would violate ¬ß1.1 layering, so `param_bus` moved from `synth-host` to `synth-engine` ‚Äî the bus is the engine's I/O ports, and crossbeam-channel / arc-swap are pure-Rust concurrency primitives not on the xtask check-deps forbid list. `synth-host/Cargo.toml` dropped the now-unused direct arc-swap + crossbeam-channel deps. `synth-ui::ToneSmithyApp` now takes the sender + snapshot slot and renders: a pitch-offset slider (-24..=24 semitones), an amp-release slider (5 ms..=3 s, logarithmic), a Sine/Saw toggle, and a live snapshot read-back showing voice_active + current values. UI requests a 33 ms repaint so voice_active updates smoothly.

##### C5 ‚Äî on-screen virtual keyboard + drop the hardcoded NoteOn

New `synth-ui::keyboard::VirtualKeyboard`: a 2-octave painted piano (white + black keys, starting at C3 / MIDI 48). `Sense::click_and_drag` on the keyboard rect; each frame the widget hit-tests the pointer (black keys first because they overlay), compares against `held_note`, and emits NoteOff(old) + NoteOn(new) on transitions. Mouse release emits NoteOff. Active key tinted blue. Single-voice mono to match the engine. The hardcoded `NoteOn(69)` in `synth-app` is gone; only the waveform-select seeding remains so the keyboard plays a saw by default.

##### C6 ‚Äî `assert_no_alloc` guard on `Engine::process_stereo` + `handle`

New integration test `crates/synth-engine/tests/no_alloc.rs` installs the `assert_no_alloc` global allocator and renders 10 seconds of 48 kHz audio in 256-frame blocks through a representative event mix (NoteOn/NoteOff, ParameterChange, SetOscillatorWaveform) inside `assert_no_alloc(|| { ... })`. The test stops at the engine boundary ‚Äî snapshot publishing intentionally allocates one Arc per block in the host per ¬ß1.4, and the recycled-pool optimisation from ¬ß2.5 is a later milestone (docstring spells this out). Added `assert_no_alloc = "1.1"` to workspace.dependencies (engine dev-dep only); `deny.toml` got `BSD-1-Clause` added to the allow list (same family as BSD-2/3 already allowed, even more permissive).

M1's "done when" ‚Äî sustained click-free notes driven by a virtual key with UI-controlled pitch + release and no audio-thread allocations ‚Äî is now structurally met. The DSP path is exercised by 17 engine unit tests + the no-alloc integration test; clippy, check-deps, and cargo-deny licenses all clean.
