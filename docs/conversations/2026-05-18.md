### [15:40:00] **You**
> there is a click sound at the start of each note

### [15:42:59] Claude (Sonnet 4.6)
Root cause: `voice.rs:47` called `self.oscillator.reset_phase()` unconditionally on every note-on. When a note is retriggered while the ADSR is still in release (level > 0), the phase snaps to 0 — for sine that forces the output to `level × sin(0) = 0`, jumping from whatever value was playing. For saw it jumps to `level × -1.0`. Both produce an audible click.

Fix: only reset the oscillator phase when the amp envelope is idle (i.e. the first note from silence). On retrigger the phase continues uninterrupted, so there is no discontinuity in the waveform output while the envelope level is non-zero. Added a regression test `retrigger_during_release_produces_no_output_discontinuity` that verifies the output step between the last release sample and first retrigger sample is below 0.05.

All 22 tests pass.

**Commits:** `5a471e7` — Fix retrigger click: skip phase reset when envelope is non-idle

### [15:51:27] Claude (Sonnet 4.6)
